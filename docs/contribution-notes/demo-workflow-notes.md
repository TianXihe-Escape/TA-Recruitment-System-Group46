# Demo Workflow Notes

## Purpose

This note explains how the sample dataset supports an end-to-end demonstration workflow.

## TA-side workflow

The demo data supports TA-side demonstration of:

- profile preparation
- CV and supporting-document references
- job browsing
- application submission
- status tracking

## MO-side workflow

The demo data supports MO-side demonstration of:

- job ownership
- applicant review
- skill comparison
- reviewer notes
- decision updates

## Admin-side workflow

The demo data supports Admin-side demonstration of:

- workload visibility
- application review visibility
- account-related oversight
- system-wide monitoring

## Why this matters

A prepared demo workflow makes the final system easier to explain and helps the team present a coherent end-to-end recruitment story.
