# Recruitment Record Notes

## Purpose

This note summarizes the main record types used in the TA Recruitment System and how they relate to each other.

## Main record types

The project works with several connected record types:

- user
- applicant profile
- job posting
- application
- allocation

## User

A user record stores account and role information. It identifies whether the person is a TA, MO, or Admin user.

## Applicant profile

An applicant profile extends the TA-side information with details such as:

- name
- programme
- year of study
- skills
- CV path

## Job posting

A job-posting record stores the recruitment opportunity created by an MO, including module, duties, required skills, hours, and deadline information.

## Application

An application links an applicant profile to a job posting. It also stores recruitment-review information such as status, reviewer notes, and match-related details.

## Allocation

An allocation record represents accepted workload assignment after a successful application outcome.

## Why these links matter

These connected records allow the system to show a complete recruitment flow instead of only isolated screens or forms.
