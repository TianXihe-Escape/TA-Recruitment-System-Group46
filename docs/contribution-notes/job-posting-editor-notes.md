# Job Posting Editor Notes

## Purpose

This note explains the role of the job-posting editor in the MO workflow.

## Main use

The job-posting editor allows a Module Organiser to create or update recruitment records for module support, invigilation, and other related activities.

## Main information entered

Typical job fields include:

- module code
- module title
- semester
- duties
- required skills
- hours
- workload type
- schedule details
- application deadline

## Why this screen matters

The quality of the job-posting data affects later parts of the recruitment workflow, including applicant review, match interpretation, and workload monitoring.

## Relationship to generated IDs

The MO user focuses on job information rather than manually managing identifiers. This keeps the editor easier to use while allowing the system to preserve structured stored records.
