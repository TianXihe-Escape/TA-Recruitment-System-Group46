# Admin Role Explanation

## Purpose

This note explains the system-level responsibilities of the Admin role.

## Main responsibilities

The Admin role provides oversight across the whole recruitment system rather than participating as a normal applicant or module owner.

## Typical Admin actions

Admin users can:

- inspect overall recruitment data
- monitor TA workloads
- review applications across modules
- manage MO accounts
- load or reset demo data
- export recruitment information

## Why this role matters

The Admin role helps the system support governance, oversight, and broader operational visibility beyond one single job or module.

## Relationship to other roles

Admin users observe and manage the wider environment in which TAs apply and MOs recruit.
